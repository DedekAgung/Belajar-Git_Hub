Branch (Cabang)
  Cabang dari sebuah repo yang berisi commit berada pada master branch ( cabang utama )

Branching 
  Percabangan yang dibuat namun tidak ke master branch namun membuat branch baru di atasnya. 
    -Berfungsi untuk membuat snapshot tanpa menggangu jalur utama (Master Branch).
    -Berfungsi juga untuk experiment fitur.
    -Berfungsi jika terdapat lebih dari 1 orang mengerjakan repo yang sama agar dapat mengerjakan 1 source code pada waktu bersamaan tanpa menggangu jalur utama.

Merge/Merging
  Menggabungkan 2 branch.
  Pull request untuk menggabukan 2 branch.
    kenapa kita harus pull request? karena untuk meminta izin pemilik master branch agar branch kita masuk/menggabungkan ke master branch.

Merge Konflik
  merge conflict adalah saat 2 branch digabungkan namun salah satu branch mengubah line yang sudah ada maka dari itu terjaci conflict/tabrakan line. dan kita pemilik 
  master branch harus resolve conflicts ( menyelesaikan masalah/tabraknya ) secara manual.
  
  

 


-garis utama pada branch itu dinamakan master branch.
